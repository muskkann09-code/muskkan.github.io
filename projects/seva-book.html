Seva Book

Digital teacher service book for viewing & managing Seva Book entries with offline support and Firebase sync.

FlutterFlow Local DB Firestore Offline-first FCM
View on Play Store

Project Overview

Seva Book is a digital teacher service log app that replaces paper diaries with an easy-to-use digital record of service/attendance entries. Built for educational institutions with privacy-first design and offline capabilities.

Key Features

  • Teacher/service record display with clean, intuitive UI
  • Local database to store records for offline viewing
  • Automatic sync with Firestore when online
  • Push notifications for announcements (FCM)
  • Privacy-first design with no third-party data sharing
  • Fast listing and search of service records
  • Export functionality for record sharing
  • User-friendly interface for non-technical users

Technical Implementation

UI & Navigation: Created responsive layouts in FlutterFlow with custom widgets for record cards and dynamic lists.

Custom Functions: Developed Dart functions to transform and filter records, including date-logic and history aggregation algorithms.

Offline-first Architecture: Implemented local database storage using SQLite patterns with conflict resolution and intelligent sync logic.

Firebase Integration: Hooked up Firebase Authentication (optional) and Cloud Firestore for remote record storage with real-time updates.

Notifications: Implemented FCM for announcements, updates, and in-app notifications with customizable channels.

Tech Stack Used

FlutterFlow Dart Custom Functions Firebase Firestore Local DB (SQLite) FCM Firebase Auth Play Console GitHub

Challenges & Solutions

Challenge: Implementing reliable offline sync with conflict resolution for multiple device usage.

Solution: Created a timestamp-based sync system with conflict detection and user prompts for resolution. Implemented background sync when network available.

Challenge: Ensuring data privacy and security for sensitive teacher records.

Solution: Implemented local encryption for stored data, secure Firebase rules, and optional biometric authentication for app access.

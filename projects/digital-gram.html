Digital Gram

Document management app with category & year-wise organization, instant search, and offline viewing capabilities.

FlutterFlow Search Indexing Firestore Offline Cache PDF Viewer
View on Play Store

Project Overview

Digital Gram organizes documents into categories and years, providing instant document viewing and text search for quick information retrieval. Built for organizations needing efficient document management on mobile devices.

Key Features

  • Category-based navigation with hierarchical organization
  • Year-wise document filtering and sorting
  • Fast in-app text search across document contents
  • Offline viewing and caching support for recently accessed documents
  • Clean, minimal UI optimized for quick lookups
  • Document metadata display and organization
  • Favorites and bookmark system for important documents
  • Share functionality for document distribution

Technical Implementation

UI & Navigation: Built intuitive UI with FlutterFlow, implementing category trees and document hierarchies.

Search System: Created custom indexing functions for text search using Dart, implementing efficient search algorithms that work offline.

Data Management: Integrated Firestore for document metadata and optional REST APIs for external content sources.

Caching Layer: Built local caching system using SQLite patterns with intelligent cache management and cleanup.

Document Viewer: Integrated PDF viewer with zoom, search within document, and bookmarking capabilities.

My Role (End-to-End Development)

  • Designed and implemented complete UI/UX in FlutterFlow with custom navigation flows
  • Created custom indexing functions for fast text search across documents
  • Built reusable Custom Widgets for document cards, lists, and viewer controls
  • Implemented REST API integrations (for external content sources) with secure handling
  • Integrated Firestore for metadata syncing and real-time updates
  • Developed offline-first local database caching and intelligent sync logic
  • Added FCM notifications for new documents and updates
  • Managed builds, Play Console listings, app signing, and release rollouts
  • Monitored post-release analytics, crash logs, and applied performance fixes

Tech Stack Used

FlutterFlow Dart Custom Functions PDF Viewer Integration Cloud Firestore REST APIs Local DB (Caching) FCM Play Console

Challenges & Solutions

Challenge: Implementing efficient search across thousands of documents without performance impact.

Solution: Created a pre-indexing system that processes documents server-side and delivers optimized search indexes to the app.

Challenge: Managing cache size for documents while ensuring important content remains available offline.

Solution: Implemented LRU (Least Recently Used) cache algorithm with user configurable cache size limits and priority settings.

E-School Documents

PDF viewer app with text search, category & year organization, offline caching, and Firebase integration.

FlutterFlow Firebase PDF Viewer Local DB FCM
View on Play Store

Project Overview

E-School Documents provides category & year-wise listing of office documents with in-app PDF viewing and full-text search. Built for quick lookup of official documents with minimal permissions and optimized for low-end devices.

Key Features

  • Category-based document browsing with intuitive navigation
  • Year-wise document organization for easy filtering
  • In-app PDF viewer with full-text search capabilities
  • Custom indexing functions for fast text search across documents
  • Offline caching using local database for reduced network calls
  • Firebase Storage for document hosting and Firestore for metadata
  • FCM push notifications for content updates
  • Lightweight UI optimized for fast load times on low-end devices

Technical Implementation

UI & Navigation: All screens and navigation flows implemented in FlutterFlow with custom widgets for consistent design.

Custom Functions: Wrote Dart functions to parse and index PDF text for fast in-app search. Created custom indexing logic that processes documents in the background.

Backend Integration: Integrated Firebase Storage for document hosting and Cloud Firestore for metadata management. Implemented secure data fetching with error handling.

Offline Capabilities: Implemented local database caching using SQLite patterns for offline access. Documents are cached on first view and available offline.

Performance Optimization: Integrated third-party PDF viewer and optimized rendering for performance. Implemented lazy loading for large document lists.

Tech Stack Used

FlutterFlow Dart (Custom Functions) Firebase Storage Cloud Firestore Local DB (SQLite) PDF Viewer Integration FCM Play Console

Challenges & Solutions

Challenge: Implementing efficient text search across large PDF files without affecting app performance.

Solution: Created a custom indexing system that processes PDFs in the background and stores search indexes locally. Implemented incremental indexing for new documents.

Challenge: Handling offline access for large document collections.

Solution: Implemented a smart caching system that prioritizes frequently accessed documents and provides partial offline access with clear user feedback.
